## Project

Autonomous task orchestrator routing work across Claude Code, Gemini CLI, and Codex CLI. Tasks come from an Obsidian vault Markdown queue (`99_System/AI/agent-queue.md`). Pure Python stdlib + pyyaml, Windows-first.

## Commands

```bash
# Run all tests (~674 tests, ~6s)
python -m pytest tests/ -q

# Run a single test file
python -m pytest tests/test_parallel_runner.py -v

# Run a single test
python -m pytest tests/test_queue_manager_regressions.py::test_extract_cwd_supports_spaces -v

# Install dependencies
pip install -r requirements.txt

# Validate setup (CLIs, vault, .env, Telegram, etc.)
python orchestrator.py --doctor
python orchestrator.py --doctor --fix --yes

# Other modes
python orchestrator.py                # single-shot: process queue once
python orchestrator.py --watch        # continuous mode with heartbeat
python orchestrator.py --dry-run      # parse queue without executing
python orchestrator.py --check-limits # show provider capacity
python orchestrator.py --list-tools   # show available #tool: handlers
python orchestrator.py --dashboard    # launch analytics web dashboard
```

## Architecture

**Execution flow**: Queue read â†’ provider selection (fallback chain) â†’ profile loading â†’ policy check â†’ skill gating â†’ memory context injection â†’ prompt building â†’ provider execution â†’ result persistence â†’ heartbeat.

Key components:
- **`orchestrator.py`**: Main loop (`run_once`/`run_watch`), prompt building (`_build_prompt`), file change tracking via before/after snapshots. `run_watch()` calls `_log_capacity()` once after startup delay so the dashboard timeline has a fresh data point from the first second. `run_watch()` starts `start_heartbeat_thread()` so scheduled checks fire during long tasks. Tool-task `error_code="timeout"` marks retry directly without provider fallback (prevents next provider failing non-retryably â†’ `[-]` finalization that would incorrectly satisfy `#needs:` deps). `run_once()` returns `None` when all tasks are blocked by `#needs:` deps â€” `run_watch()` then sleeps `SLEEP_POLL_INTERVAL` (5 min) instead of 30s busy-loop, and skips `notify_queue_complete()` to avoid Telegram spam
- **`dispatcher.py`**: Provider selection with fallback chain (Claude â†’ Gemini â†’ Codex), cooldown management, provider-bound model aliases for all three CLIs: Claude (`#claude_haiku`, `#claude_sonnet`, `#claude_opus`), Gemini (`#gemini_pro`, `#gemini_flash`, `#gemini_flash_lite`), Codex (`#codex_5` â†’ gpt-5.5 flagship, `#codex_5_4` â†’ gpt-5.4, `#codex_mini` â†’ gpt-5.4-mini). Without a tag the CLI's own default model is used. `get_provider_by_name()` for cross-provider tool access
- **`queue_manager.py`**: Obsidian MD queue parsing with sidecar `.lock` file locking (msvcrt on Windows, fcntl on Unix). Regex-based metadata extraction (`cwd:`, `#tool:`, `#agent:`, `#parallel`, `#claude_*`, `#id:`, `#needs:`, `#pass1:`, `#pass2:`, etc.). `extract_pass_providers()` returns `{1: "claude", 2: "gemini"}` for cross-provider tool support. UTF-8 with cp1252 fallback. Smart wikilink/file context injection with TF-IDF section extraction. `_parse_subtask_line()` shared helper used by both `read_queue_items()` and `_replace_open_task_line()`. Subtask-aware task matching in queue mutations (mark_done/mark_retry/finalize) prevents wrong-task collisions in parallel queues. Two-pass dependency resolution in `read_queue_items()`: Pass 1 collects open tasks, Pass 2 resolves `#needs:` deps against completed IDs (`_collect_completed_ids()` scans `[x]`/`[-]` lines). Blocked tasks get `QueueTask.blocked_reason != ""` and are skipped by `run_once()` without marking done.
- **`providers/base.py`**: `BaseProvider` ABC with per-provider `_lock` for cooldown state and `threading.local()` for per-thread forced model. `RunResult` includes `input_tokens`/`output_tokens` for capacity estimation **plus `cache_creation_input_tokens`/`cache_read_input_tokens`** (Anthropic prompt-cache fields, populated only by Claude provider â€” used for billing analytics, NOT for 5h/7d quota gates). Class-level `supports_sessions: bool = False` capability flag â€” Claude overrides to True; Codex/Gemini stay False. `run()` accepts `session_id` and `resume` parameters that providers without session support silently ignore
- **`providers/claude.py`**: Uses `--output-format json` to capture actual token usage + `--exclude-dynamic-system-prompt-sections` to move per-machine sections (cwd, env, git status) from system-prompt into the first user message â†’ static system-prompt enables Anthropic 1h prompt-cache hits across sequential subprocess calls. `--allowedTools` always includes `Task` (so deep-security-audit's subagent-mode can fan out to 6 personas via Claude's internal Task-tool). `_parse_json_response()` returns `(output_text, token_dict)` with all four token fields. When `session_id` is set AND `config.CLAUDE_SESSION_ENABLED=True`: appends `--session-id <uuid>` (first call) or `--resume <uuid>` (subsequent calls) â€” caller tracks which is which. `--resume` against a non-existent UUID returns typed `error="session_missing"` (substring match on stderr "No conversation found with session ID") so tools can fall back to a fresh session. `supports_sessions = True`. Passes `--model <id>` when `_forced_model` is set
- **`providers/gemini.py`**: Uses `--yolo` for full tool access, `--approval-mode default` for read-only. Passes `--model <id>` (e.g. `gemini-3-pro-preview`, `gemini-3-flash-preview`) when `_forced_model` is set â€” prevents silent fallback to older Gemini tiers
- **`providers/codex.py`**: Uses `codex exec --full-auto` or read-only sandbox. Passes `--model <id>` (e.g. `gpt-5.4-mini`) when `_forced_model` is set
- **`providers/openrouter.py`**: HTTP-based provider (OpenAI-compatible chat completions, `urllib.request` stdlib â€” no `requests` dep). Pay-per-token, never in fallback chain â€” only activated via explicit `#openrouter` or `#or_*` tag. Conditionally registered in `dispatcher._providers` ONLY when `OPENROUTER_API_KEY` env var is set; otherwise tagged tasks fall through to the default chain (claude/gemini/codex). `supports_sessions = False`. Token extraction via `usage.{prompt_tokens, completion_tokens, prompt_tokens_details.cached_tokens}` â†’ `RunResult.{input_tokens, output_tokens, cache_read_input_tokens}`. Error mapping: HTTP 429 â†’ `rate_limit` + cooldown, 401/403 â†’ `auth_error` (no cooldown, config issue), 5xx/URLError â†’ `unreachable`, JSON parse fail â†’ `parse_error`, 200-with-error-body â†’ `api_error`. `read_only` accepted but no-op (provider has no tool execution surface). Designed for single-call non-agentic tasks (heartbeat LLM-check, memory summaries, telegram `/chat`) â€” DO NOT route agentic tools (dev-loop, review-loop, deep-security-audit) here. `dispatcher._limits_ok("openrouter", ...)` is a hardcoded `True` since pay-per-token has no subscription quota tracked by cclimits
- **`policy.py`**: `PolicyEngine` singleton â€” AUTO/APPROVE/DENY classification from vault YAML, blocks on `threading.Event` for Telegram approval
- **`usage_suggester.py`**: `UsageSuggester` singleton â€” proactive task suggestions when provider capacity is underutilized. Same threading pattern as PolicyEngine
- **`memory.py`**: TF-IDF + temporal decay search over past task results stored in vault. Auto-archival after 180 days. `append_lesson()`/`get_lessons_context()` re-enabled with LLM-generated summaries for tool loops (review-loop, dev-loop) to capture generalized patterns and root causes. `get_lessons_context(tool_name, cwd)`: CWD-filtered injection â€” `| * |` entries are universal (always injected), project-specific entries only when CWD matches. `append_lesson(universal=True)` writes `*` as project. Semantic dedup via TF-IDF similarity (`_LESSON_DEDUP_THRESHOLD=0.45`) at write time prevents redundant entries. `get_context_for_task()`: CWD-preference â€” wenn â‰Ą2 same-CWD Treffer ĂĽber Threshold, werden cross-project Matches verworfen. `append_daily_log()`: 80-Zeichen First-Line-Summary (statt 300 chars).
- **`heartbeat.py`**: Scheduled health checks with mtime-reloading config from vault `HEARTBEAT.md`. 10 built-in handlers: queue-idle, git-status, disk-space, check-limits, log-capacity, summarize, stale-branch, usage-suggest, session-cleanup, model-check. `_INTERVAL_RE` accepts `## Every N minutes/hours/days` (days = NĂ—1440 minutes) so monthly cadences work via `## Every 30 days`. `## Every 0 X` sections are dropped at parse time with a warning (without that the items would silently inherit `interval_min=0` and be treated as daily). `_check_session_cleanup()` reads the sidecar registry (`session_registry.py`), prunes entries older than `ORCH_SESSION_RETENTION_DAYS=14`, and deletes matching `~/.claude/projects/**/<uuid>.jsonl` files via glob â€” interactive Claude Code sessions are NEVER touched because they're not in the registry. `_check_model_updates()` runs Phase A (CLI-Probe via `_probe_model`: minimal `read_only=True` ping per alias; SKIPS providers in cooldown to avoid stacking errors; transient keywords `_PROBE_TRANSIENT_KEYWORDS` cover BOTH typed errors `rate_limit` AND raw stderr `rate limit` plus auth/login/credential/expired/token; dead keywords `_PROBE_DEAD_KEYWORDS` are narrow phrases like "model not found", "unknown model", "no such model" â€” avoids false positives like "unsupported encoding") + Phase B (LLM-Heuristik via `_llm_check_for_newer_models`: TWO-TIER provider selection â€” **Tier 1**: OpenRouter (free model, `$0/call`) if `OPENROUTER_API_KEY` is configured and not in cooldown; **Tier 2**: existing default-chain fallback via `select_provider(exclude={"openrouter"})` when OpenRouter fails or is unavailable. Single prompt with TODAY'S DATE as anchor; the prompt now includes ALL provider aliases including the OpenRouter ones so model-drift in `OPENROUTER_MODEL_ALIASES` is caught too. Strict OK match `text.upper().rstrip(".").strip() == "OK"` rejects "Okay,..." prefix; LLM-call FAILURE returns visible `âš ď¸Ź LLM-Check failed: <error>` instead of being silently swallowed). Returns `None` when everything is current â†’ no Telegram spam. **Persistent state** (`logs/heartbeat-state.json`): items with `interval_min >= 1440` (â‰Ą 1 day) survive orchestrator restarts so a `## Every 30 days` model-check doesn't fire on every `--watch` start (cost saving). `HeartbeatRunner._lock` prevents concurrent `run_due()` calls (non-blocking acquire â†’ skip if already running). `start_heartbeat_thread(heartbeat, queue_read_fn, stop_event, poll_sec=60)` launches a daemon thread so checks fire on time even when the main thread is blocked for hours inside a long task
- **`analytics.py`**: Parses task results, log files, queue events into `TaskRecord`/`LimitSnapshot`/`QueueEvent` dataclasses. `TaskRecord` includes the four token fields (input/output + cache_creation/cache_read) parsed from frontmatter. Aggregation functions + `get_dashboard_data()` with 30s TTL cache. `_billing_cost_units(records)` returns weighted token-units (inputĂ—1, cache_creationĂ—1.25, cache_readĂ—0.1, outputĂ—5 â€” Anthropic billing weights, NOT quota weights) for cost-trend visualization; `_cache_hit_rate(records)` returns cache_read / (input + cache_creation + cache_read) as percentage. Both NOT used for quota gating (limits.py owns that). `_get_current_limits()` reads the most-recent-per-provider snapshot from `logs/capacity-log.md`; result exposed as `current_limits` in dashboard API. Dashboard data exposes `billing_recent`/`billing_total`/`cache_hit_rate_recent`/`cache_hit_rate_total`. Queue events read from `logs/queue-events.log`
- **`dashboard.py`**: Standalone HTTP server (port 8411) serving Chart.js dashboard. `GET /` HTML, `GET /api/data` JSON. Auto-refresh 60s. Also launchable via `--dashboard` flag
- **`scripts/safety_hook.py`**: Claude Code `PreToolUse` hook â€” hard-denies dangerous Bash commands using `SAFETY_DENY_PATTERNS` from config.py. Works even in `--dangerously-skip-permissions` mode. Registered in `.claude/settings.local.json`

- **`limits.py`**: `cclimits` wrapper for provider capacity checks, OAuth refresh handling. Direct `cclimits` invocation (no npx); disk-cache via `--cache-ttl 600` limits API calls to ~6/h. **Polling-Intervalle**: `_BG_POLL_IDLE_SEC=3600` (1h, queue empty) â€” `set_queue_idle(False)` wakes the bg thread on task arrival so the cache is fresh before the next task starts; the hourly idle poll only keeps the dashboard timeline warm. `_BG_POLL_AVAILABLE_SEC=300` (5 min, queue active) â€” covers within-batch drift between consecutive tasks in run_once; rate_limit errors trigger `get_limits(force_refresh=True)` synchronously, so cache staleness can never delay error recovery. `set_paused(True/False)`: while paused the bg refresh thread skips all cclimits calls and only wakes on resume â€” `run_watch()` calls this around its `/pause` wait so the orchestrator stops burning the monitoring quota during user-pauses. - **`run_orchestrator.ps1`**: Crash-resistant watchdog wrapper. Compatible with both Windows PowerShell 5.1 and PowerShell 7+ â€” uses `Where-Object` for crash-window filtering instead of the `[System.Predicate[datetime]]` ScriptBlock-cast (which only works in PS 7+). Restarts `python orchestrator.py --watch` on non-zero exit, exponential backoff (10s â†’ 5min cap), crash-loop guard (â‰Ą5 crashes / 10 min â†’ 30 min cooldown + Telegram alert). Reads `TELEGRAM_BOT_TOKEN`/`TELEGRAM_CHAT_ID` from `.env` for restart notifications via `Invoke-RestMethod` (no Python dependency for alerting). `Get-EnvValue` matches Python's `_normalize_dotenv_value` rules: strips UTF-8 BOM on first line, requires whitespace before `#` for unquoted-value comments (so URLs/hashes containing `#` survive), supports quoted values with optional trailing comments. - **`tools/base_tool.py`**: `BaseTool` ABC + `_build_system_prompt(provider_name, memory_context, tool_name, cwd)` assembles system prompt with 4 memory layers ordered by stability for max prompt-cache reuse: (1) provider system prompt (SOUL+SAFETY, static), (2) curated MEMORY.md, (3) lessons (cwd-filtered, stable per tool+cwd), (4) daily log (grows during the day), (5) task-specific TF-IDF memory_context. **Lessons before Daily** is intentional â€” daily grows over the day and would otherwise break the prefix cache for tool+cwd reruns. `ToolResult` carries the same four token fields as `RunResult`. `TokenCounter` helper (`.add(result)` / `.as_kwargs()`) aggregates tokens across phases â€” used by review-loop and dev-loop, other tools may adopt incrementally. **`SessionContext`** helper (`SessionContext.create(provider, tool_name, cwd, cap=5)` + `.first_call_kwargs()` / `.resume_kwargs()` / `.bump()` / `.needs_rollover()` / `.rollover()`) manages CLI session lifecycle for tools that share conversation history across phases. Activates only when `provider.supports_sessions and config.CLAUDE_SESSION_ENABLED`; otherwise transparently returns empty kwargs (callers fall back to today's stateless pattern). Auto-registers UUIDs in the sidecar registry. All tool subclasses pass `tool_name=self.name, cwd=cwd` for CWD-filtered lesson injection
- **`session_registry.py`**: Append-only JSONL whitelist of orchestrator-created Claude session UUIDs at `logs/orchestrator-sessions.jsonl`. `register_session(uuid, tool, cwd)` appends an entry; `is_orchestrator_session(uuid)` checks membership; `prune_old(retention_days)` splits into kept/expired and rewrites the registry. - **`tools/research_qa.py`**: `ResearchQATool` â€” 3-phase read-only pre-implementation workflow (Discovery â†’ Analysis â†’ Questions). Output to `{cwd}/.research-qa/`. Registered as `#tool:research-qa`
- **`tools/review_loop.py`**: `ReviewLoopTool` â€” iterative code review fixing ALL P1/P2/P3 findings (max 20 iterations). Infinite-loop detection via finding signature dedup. - **`tools/dev_loop.py`**: `DevLoopTool` â€” Research+Plan â†’ Execute â†’ Quality-Review + Resolution-Review (`RESOLVED`/`PARTIAL`/`UNRESOLVED`) loop until both reviews pass (max 20 iterations). **Phase 1 (Research) and Phase 1.5 (Plan) are merged into a single subprocess call** (`_RESEARCH_AND_PLAN_PROMPT` produces both sections; saves ~42 k cache_creation tokens vs. the prior two-call pattern). Fallback on `error="session_missing"` (e.g. cleanup deleted JSONL between calls): rollover + retry once with the same prompt. Registered as `#tool:dev-loop`
- **`tools/critical_review.py`**: `CriticalReviewTool` â€” 3-pass adversarial review. Pass 1: radical-honesty analysis. Pass 2: adversarial challenge by different persona. Pass 3 (Synthesis): produces improved plan as `{name}-v2.md` (only when plan file referenced in task). Plan file resolution: file paths in CWD â†’ wikilinks in vault. - **`tools/security_audit.py`**: `SecurityAuditTool` â€” 2-phase security audit + fix workflow. - **`tools/deep_security_audit.py`**: `DeepSecurityAuditTool` â€” multi-agent deep security audit with 6 expert personas + CISO synthesis + optional fix. **Phase C1 capability switch** (`run()` â†’ `_run_subagent_mode` vs. `_run_sequential_mode`): when `provider.supports_sessions` AND `CLAUDE_SESSION_ENABLED`, a single Claude subprocess spawns the 6 personas IN PARALLEL via the Task tool, writes per-persona reports + the CISO synthesis as files via the Write tool, and returns a brief executive summary as the final assistant message. Phase 7: CISO synthesis deduplicates, cross-validates, builds attack chains, creates prioritized remediation roadmap. Phase 8 (optional): fix implementation. `#no-fix` tag skips fix phase. ## Phase B Feature Flag â€” `CLAUDE_SESSION_ENABLED`

CLI-level conversation sessions on the Claude provider are gated by `config.CLAUDE_SESSION_ENABLED` (default **OFF**, opt-in via `.env`). When enabled:
- Tools that allocate a `SessionContext` get a UUID and pass `--session-id`/`--resume` on subsequent `claude --print` calls.
- Anthropic prompt-cache hits go from "static system-prompt only" to "full conversation history" â€” projected ~30-50 % token savings on multi-phase tools (dev-loop, review-loop).
- All UUIDs are registered in `logs/orchestrator-sessions.jsonl` (sidecar) so the heartbeat session-cleanup never deletes interactive Claude Code sessions.
- Fresh sessions every `cap=5` iterations cap conversation length (Sonnet 200 k context limit, Opus 1 M).
- On `error="session_missing"` (e.g. cleanup race) tools rollover and retry once.

**Rollback**: set `CLAUDE_SESSION_ENABLED=false` in `.env` and restart the orchestrator. ClaudeProvider then ignores `session_id`/`resume` parameters and falls back to today's stateless `claude --print` invocation. No code revert needed.

`config.ORCH_SESSION_RETENTION_DAYS=14` controls how long expired registry entries live before the cleanup deletes them.

Codex and Gemini providers also have CLI-level resume capabilities, but `supports_sessions=False` keeps them on stateless behaviour until empirical evidence justifies the implementation cost.

## Key Patterns

- **Singletons with threading**: `PolicyEngine`, `UsageSuggester`, providers â€” each has own `_lock`, `threading.Event` for blocking operations, no global mutex
- **Provider-bound model tags**: `#claude_opus`, `#gemini_flash`, `#codex_mini`, `#or_minimax_free`, `#or_glm` etc. are resolved via `config.model_id_for_provider(tag, provider_name)`, which returns `None` if the tag doesn't belong to the target provider. This prevents a Claude model ID from leaking to Gemini during provider fallback. The `_forced_model` attribute lives on `BaseProvider` (via `threading.local()`), applied to any provider. Orchestrator and `parallel_runner` set it per-task without a claude-only gate. Cross-provider tools (e.g. `critical_review.py` pass2) re-apply the tag against the pass2 provider separately â€” see `CriticalReviewTool.run()` around `pass2_provider.run(...)`.
- **OpenRouter never in fallback chain**: `dispatcher._PRIORITY` deliberately omits `openrouter` so untagged tasks never accidentally route to a pay-per-token provider. Tag resolution uses `_providers.get(name)` (not `[name]`) so an `#or_*` tag without `OPENROUTER_API_KEY` configured silently falls back to claude/gemini/codex instead of raising KeyError. Same pattern would extend cleanly to future opt-in providers.
- **Mtime-cached config**: Policy, profiles, SOUL.md, heartbeat all use `(mtime, content)` tuple caching for hot-reload in `--watch` mode
- **Token-budget injection**: `_build_prompt()` truncates skill/memory/wikilink context to `PROMPT_*_TOKENS` constants before assembly
- **Sidecar file locking**: `queue_manager.py` uses `.lock` file with platform-specific locking for multi-process safety
- **Subtask-aware queue mutations**: `mark_done/mark_retry/finalize_task_with_result` accept `subtasks` kwarg; `_replace_open_task_line` uses it to disambiguate duplicate task texts in parallel queues. Fallback re-scan is O(N) â€” skips subtask block scan for non-matching task lines.
- **`task_subtasks` in run_once**: Extracted via `getattr(queue_task, "subtasks", None)` at loop start for test-mock compatibility (some tests use bare `SimpleNamespace`).
- **Task dependencies (`#id:`/`#needs:`)**: `#id:name` tags a task with a unique ID. `#needs:name1,name2` blocks a task until all named deps appear as `[x]` or `[-]` in the file. `_collect_completed_ids()` scans the full file for done/failed tasks. Two-pass in `read_queue_items()` â€” short-circuit if no `#needs:` present. Blocked tasks keep `QueueTask.blocked_reason != ""` and are skipped by `run_once()` (no `mark_done` â†’ stays in queue for next cycle). Queue header shows `(N ausfĂĽhrbar, M blockiert)` when any tasks are blocked.
- **`.env` comment stripping**: `_normalize_dotenv_value()` requires whitespace before `#` for unquoted values (protects URLs/paths containing `#`); quoted values allow `#` anywhere after the closing quote.
- **HTTP 429 resilience**: When `cclimits` monitoring API returns 429, `limits.py` retries with backoff (5s/10s), then applies 3-tier fallback: (0) local JSONL via `claude-monitor` (`_get_claude_limits_from_local`, `CLAUDE_PLAN` env var, uses `token_counts.total_tokens`), (1) snapshot cache with estimated usage tracking (`_429_base_snapshot`, `_429_estimated_usage`), (2) optimistic cold-start. Polls back off to 5 minutes. `report_estimated_usage()` called after each task. State resets when 429 clears. Disk-cache (`--cache-ttl 600`) reduces normal API calls.
- **3-tier token estimation**: `estimate_task_usage_pct()` in `limits.py` â€” (1) actual token counts from Claude JSON output, (2) text-based estimate from prompt/output char lengths, (3) duration heuristic fallback. Configured per provider via `ESTIMATE_TOKENS_PER_PCT`.

## Safety Rules (enforced in code)

- **Hard deny (Claude Code hook)**: `scripts/safety_hook.py` as `PreToolUse` hook blocks dangerous Bash commands before execution â€” works even in `--dangerously-skip-permissions` mode. Patterns defined in `config.SAFETY_DENY_PATTERNS`
- **Soft deny (prompt injection)**: `SAFETY_RULES` text (auto-generated from same patterns) injected into all provider prompts via `SYSTEM_PROMPTS` â€” covers Gemini/Codex which have no hook system
- **Blocked categories**: `rm -rf`, `git push --force/-f`, `git reset --hard`, `git clean -f`, `git checkout -- .`, `DROP/TRUNCATE TABLE`, `DELETE FROM` without WHERE, `format`/`mkfs`/`diskpart`, fork bombs, raw disk writes, credential exfiltration via curl/wget, Windows `del /s`, `rd /s /q`, `Remove-Item -Recurse -Force`
- CWD validation against `ALLOWED_CWD_ROOTS` â€” rejects relative paths and parent escapes
- Skill gating checks requirements (bins, env vars, OS, provider) before execution
- Policy layer can block tasks pending Telegram approval
