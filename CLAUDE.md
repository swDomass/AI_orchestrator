## Project

Autonomous task orchestrator routing work across Claude Code, Gemini CLI, and Codex CLI. Tasks come from an Obsidian vault Markdown queue (`99_System/AI/agent-queue.md`). Pure Python stdlib + pyyaml, Windows-first.

## Commands

```bash
# Run all tests (~674 tests, ~6s)
python -m pytest tests/ -q

# Run a single test file
python -m pytest tests/test_parallel_runner.py -v

# Run a single test
python -m pytest tests/test_queue_manager_regressions.py::test_extract_cwd_supports_spaces -v

# Install dependencies
pip install -r requirements.txt

# Validate setup (CLIs, vault, .env, Telegram, etc.)
python orchestrator.py --doctor
python orchestrator.py --doctor --fix --yes

# Other modes
python orchestrator.py                # single-shot: process queue once
python orchestrator.py --watch        # continuous mode with heartbeat
python orchestrator.py --dry-run      # parse queue without executing
python orchestrator.py --check-limits # show provider capacity
python orchestrator.py --list-tools   # show available #tool: handlers
python orchestrator.py --dashboard    # launch analytics web dashboard
```

## Architecture

**Execution flow**: Queue read ŌåÆ provider selection (fallback chain) ŌåÆ profile loading ŌåÆ policy check ŌåÆ skill gating ŌåÆ memory context injection ŌåÆ prompt building ŌåÆ provider execution ŌåÆ result persistence ŌåÆ heartbeat.

Key components:
- **`orchestrator.py`**: Main loop (`run_once`/`run_watch`), prompt building (`_build_prompt`), file change tracking via before/after snapshots. `run_watch()` calls `_log_capacity()` once after startup delay so the dashboard timeline has a fresh data point from the first second. `run_watch()` starts `start_heartbeat_thread()` so scheduled checks fire during long tasks. Tool-task `error_code="timeout"` marks retry directly without provider fallback (prevents next provider failing non-retryably ŌåÆ `[-]` finalization that would incorrectly satisfy `#needs:` deps). `run_once()` returns `None` when all tasks are blocked by `#needs:` deps ŌĆö `run_watch()` then sleeps `SLEEP_POLL_INTERVAL` (5 min) instead of 30s busy-loop, and skips `notify_queue_complete()` to avoid Telegram spam
- **`dispatcher.py`**: Provider selection with fallback chain (Claude ŌåÆ Gemini ŌåÆ Codex), cooldown management, provider-bound model aliases for all three CLIs: Claude (`#claude_haiku`, `#claude_sonnet`, `#claude_opus`), Gemini (`#gemini_pro`, `#gemini_flash`, `#gemini_flash_lite`), Codex (`#codex_5` ŌåÆ gpt-5.5 flagship, `#codex_5_4` ŌåÆ gpt-5.4, `#codex_mini` ŌåÆ gpt-5.4-mini). Without a tag the CLI's own default model is used. `get_provider_by_name()` for cross-provider tool access
- **`queue_manager.py`**: Obsidian MD queue parsing with sidecar `.lock` file locking (msvcrt on Windows, fcntl on Unix). Regex-based metadata extraction (`cwd:`, `#tool:`, `#agent:`, `#parallel`, `#claude_*`, `#id:`, `#needs:`, `#pass1:`, `#pass2:`, etc.). `extract_pass_providers()` returns `{1: "claude", 2: "gemini"}` for cross-provider tool support. UTF-8 with cp1252 fallback. Smart wikilink/file context injection with TF-IDF section extraction. `_parse_subtask_line()` shared helper used by both `read_queue_items()` and `_replace_open_task_line()`. Subtask-aware task matching in queue mutations (mark_done/mark_retry/finalize) prevents wrong-task collisions in parallel queues. Two-pass dependency resolution in `read_queue_items()`: Pass 1 collects open tasks, Pass 2 resolves `#needs:` deps against completed IDs (`_collect_completed_ids()` scans `[x]`/`[-]` lines). Blocked tasks get `QueueTask.blocked_reason != ""` and are skipped by `run_once()` without marking done.
- **`providers/base.py`**: `BaseProvider` ABC with per-provider `_lock` for cooldown state and `threading.local()` for per-thread forced model. `RunResult` includes `input_tokens`/`output_tokens` for capacity estimation **plus `cache_creation_input_tokens`/`cache_read_input_tokens`** (Anthropic prompt-cache fields, populated only by Claude provider ŌĆö used for billing analytics, NOT for 5h/7d quota gates). Class-level `supports_sessions: bool = False` capability flag ŌĆö Claude overrides to True; Codex/Gemini stay False. `run()` accepts `session_id` and `resume` parameters that providers without session support silently ignore
- **`providers/claude.py`**: Uses `--output-format json` to capture actual token usage + `--exclude-dynamic-system-prompt-sections` to move per-machine sections (cwd, env, git status) from system-prompt into the first user message ŌåÆ static system-prompt enables Anthropic 1h prompt-cache hits across sequential subprocess calls. `--allowedTools` always includes `Task` (so deep-security-audit's subagent-mode can fan out to 6 personas via Claude's internal Task-tool). `_parse_json_response()` returns `(output_text, token_dict)` with all four token fields. When `session_id` is set AND `config.CLAUDE_SESSION_ENABLED=True`: appends `--session-id <uuid>` (first call) or `--resume <uuid>` (subsequent calls) ŌĆö caller tracks which is which. `--resume` against a non-existent UUID returns typed `error="session_missing"` (substring match on stderr "No conversation found with session ID") so tools can fall back to a fresh session. `supports_sessions = True`. Passes `--model <id>` when `_forced_model` is set
- **`providers/gemini.py`**: Uses `--yolo` for full tool access, `--approval-mode default` for read-only. Passes `--model <id>` (e.g. `gemini-3-pro-preview`, `gemini-3-flash-preview`) when `_forced_model` is set ŌĆö prevents silent fallback to older Gemini tiers
- **`providers/codex.py`**: Uses `codex exec --full-auto` or read-only sandbox. Passes `--model <id>` (e.g. `gpt-5.4-mini`) when `_forced_model` is set
- **`providers/openrouter.py`**: HTTP-based provider (OpenAI-compatible chat completions, `urllib.request` stdlib ŌĆö no `requests` dep). Pay-per-token, never in fallback chain ŌĆö only activated via explicit `#openrouter` or `#or_*` tag. Conditionally registered in `dispatcher._providers` ONLY when `OPENROUTER_API_KEY` env var is set; otherwise tagged tasks fall through to the default chain (claude/gemini/codex). `supports_sessions = False`. Token extraction via `usage.{prompt_tokens, completion_tokens, prompt_tokens_details.cached_tokens}` ŌåÆ `RunResult.{input_tokens, output_tokens, cache_read_input_tokens}`. Error mapping: HTTP 429 ŌåÆ `rate_limit` + cooldown, 401/403 ŌåÆ `auth_error` (no cooldown, config issue), 5xx/URLError ŌåÆ `unreachable`, JSON parse fail ŌåÆ `parse_error`, 200-with-error-body ŌåÆ `api_error`. `read_only` accepted but no-op (provider has no tool execution surface). Designed for single-call non-agentic tasks (heartbeat LLM-check, memory summaries, telegram `/chat`) ŌĆö DO NOT route agentic tools (dev-loop, review-loop, deep-security-audit) here. `dispatcher._limits_ok("openrouter", ...)` is a hardcoded `True` since pay-per-token has no subscription quota tracked by cclimits
- **`policy.py`**: `PolicyEngine` singleton ŌĆö AUTO/APPROVE/DENY classification from vault YAML, blocks on `threading.Event` for Telegram approval
- **`usage_suggester.py`**: `UsageSuggester` singleton ŌĆö proactive task suggestions when provider capacity is underutilized. Same threading pattern as PolicyEngine
- **`memory.py`**: TF-IDF + temporal decay search over past task results stored in vault. Auto-archival after 180 days. `append_lesson()`/`get_lessons_context()` re-enabled with LLM-generated summaries for tool loops (review-loop, dev-loop) to capture generalized patterns and root causes. `get_lessons_context(tool_name, cwd)`: CWD-filtered injection ŌĆö `| * |` entries are universal (always injected), project-specific entries only when CWD matches. `append_lesson(universal=True)` writes `*` as project. Semantic dedup via TF-IDF similarity (`_LESSON_DEDUP_THRESHOLD=0.45`) at write time prevents redundant entries. `get_context_for_task()`: CWD-preference ŌĆö wenn Ōēź2 same-CWD Treffer ├╝ber Threshold, werden cross-project Matches verworfen. `append_daily_log()`: 80-Zeichen First-Line-Summary (statt 300 chars).
- **`heartbeat.py`**: Scheduled health checks with mtime-reloading config from vault `HEARTBEAT.md`. 10 built-in handlers: queue-idle, git-status, disk-space, check-limits, log-capacity, summarize, stale-branch, usage-suggest, session-cleanup, model-check. `_INTERVAL_RE` accepts `## Every N minutes/hours/days` (days = N├Ś1440 minutes) so monthly cadences work via `## Every 30 days`. `## Every 0 X` sections are dropped at parse time with a warning (without that the items would silently inherit `interval_min=0` and be treated as daily). `_check_session_cleanup()` reads the sidecar registry (`session_registry.py`), prunes entries older than `ORCH_SESSION_RETENTION_DAYS=14`, and deletes matching `~/.claude/projects/**/<uuid>.jsonl` files via glob ŌĆö interactive Claude Code sessions are NEVER touched because they're not in the registry. `_check_model_updates()` runs Phase A (CLI-Probe via `_probe_model`: minimal `read_only=True` ping per alias; SKIPS providers in cooldown to avoid stacking errors; transient keywords `_PROBE_TRANSIENT_KEYWORDS` cover BOTH typed errors `rate_limit` AND raw stderr `rate limit` plus auth/login/credential/expired/token; dead keywords `_PROBE_DEAD_KEYWORDS` are narrow phrases like "model not found", "unknown model", "no such model" ŌĆö avoids false positives like "unsupported encoding") + Phase B (LLM-Heuristik via `_llm_check_for_newer_models`: TWO-TIER provider selection ŌĆö **Tier 1**: OpenRouter (free model, `$0/call`) if `OPENROUTER_API_KEY` is configured and not in cooldown; **Tier 2**: existing default-chain fallback via `select_provider(exclude={"openrouter"})` when OpenRouter fails or is unavailable. Single prompt with TODAY'S DATE as anchor; the prompt now includes ALL provider aliases including the OpenRouter ones so model-drift in `OPENROUTER_MODEL_ALIASES` is caught too. Strict OK match `text.upper().rstrip(".").strip() == "OK"` rejects "Okay,..." prefix; LLM-call FAILURE returns visible `ŌÜĀ’ĖÅ LLM-Check failed: <error>` instead of being silently swallowed). Returns `None` when everything is current ŌåÆ no Telegram spam. **Persistent state** (`logs/heartbeat-state.json`): items with `interval_min >= 1440` (Ōēź 1 day) survive orchestrator restarts so a `## Every 30 days` model-check doesn't fire on every `--watch` start (cost saving). `HeartbeatRunner._lock` prevents concurrent `run_due()` calls (non-blocking acquire ŌåÆ skip if already running). `start_heartbeat_thread(heartbeat, queue_read_fn, stop_event, poll_sec=60)` launches a daemon thread so checks fire on time even when the main thread is blocked for hours inside a long task
- **`analytics.py`**: Parses task results, log files, queue events into `TaskRecord`/`LimitSnapshot`/`QueueEvent`/`ToolTraceEvent` dataclasses. `TaskRecord` includes the four token fields (input/output + cache_creation/cache_read) parsed from frontmatter. Aggregation functions + `get_dashboard_data()` with 30s TTL cache. `_billing_cost_units(records)` returns weighted token-units (input├Ś1, cache_creation├Ś1.25, cache_read├Ś0.1, output├Ś5 ŌĆö Anthropic billing weights, NOT quota weights) for cost-trend visualization; `_cache_hit_rate(records)` returns cache_read / (input + cache_creation + cache_read) as percentage. Both NOT used for quota gating (limits.py owns that). `_get_current_limits()` reads the most-recent-per-provider snapshot from `logs/capacity-log.md`; result exposed as `current_limits` in dashboard API. Dashboard data exposes `billing_recent`/`billing_total`/`cache_hit_rate_recent`/`cache_hit_rate_total`. Queue events read from `logs/queue-events.log`. **`_parse_tool_traces(allowed_roots, max_age_days=30, max_events_per_file=1000)`**: globs `<root>/**/.<tool>/traces/*.jsonl` over `ALLOWED_CWD_ROOTS`, skips files older than 30 days, caps events per file. Tolerates invalid JSON lines (skip + continue). **`_tool_trace_stats(events)`**: aggregates per-tool `{runs, completed_runs, success_runs, avg_duration_sec, total_events}`. A run is "completed" if its run_id has a `run_end` event; "success" when `details.success is True`. Exposed as `tool_trace_stats` in dashboard data ŌĆö absence of `run_end` for a run_id signals an incomplete/crashed run (analytics feature, not bug).
- **`dashboard.py`**: Standalone HTTP server (port 8411) serving Chart.js dashboard. `GET /` HTML, `GET /api/data` JSON. Auto-refresh 60s. Also launchable via `--dashboard` flag
- **`scripts/safety_hook.py`**: Claude Code `PreToolUse` hook ŌĆö hard-denies dangerous Bash commands using `SAFETY_DENY_PATTERNS` from config.py. Works even in `--dangerously-skip-permissions` mode. Registered in `.claude/settings.local.json`
- **`config.py`**: Centralized constants (~70+), `.env` loader (no external dotenv), mtime-cached `SOUL.md` personality loader, provider-bound model aliases: `CLAUDE_MODEL_ALIASES`, `GEMINI_MODEL_ALIASES` (`gemini_pro` ŌåÆ `gemini-3.1-pro-preview`, `gemini_flash` ŌåÆ `gemini-3-flash-preview`, `gemini_flash_lite` ŌåÆ `gemini-3.1-flash-lite-preview`), `CODEX_MODEL_ALIASES` (`codex_5` ŌåÆ `gpt-5.5`, `codex_5_4` ŌåÆ `gpt-5.4`, `codex_mini` ŌåÆ `gpt-5.4-mini`), `OPENROUTER_MODEL_ALIASES` (4 free: `or_minimax_free` ŌåÆ `minimax/minimax-m2.5:free`, `or_deepseek_free`, `or_qwen_free`, `or_nemotron_free`; 5 paid flagships from Chinese open-source families: `or_glm` ŌåÆ `z-ai/glm-5`, `or_kimi` ŌåÆ `moonshotai/kimi-k2.6`, `or_qwen` ŌåÆ `qwen/qwen3-max`, `or_deepseek` ŌåÆ `deepseek/deepseek-v4-pro`, `or_minimax` ŌåÆ `minimax/minimax-m2.7` ŌĆö IDs verified against OpenRouter `/models` API). Env vars: `OPENROUTER_API_KEY` (required for provider registration), `OPENROUTER_BASE_URL` (default `https://openrouter.ai/api/v1`), `OPENROUTER_DEFAULT_MODEL` (used when no `_forced_model` is set). `model_id_for_provider(tag, provider_name)` resolves only if tag matches its owning provider (prevents cross-provider model ID leakage during fallback). `is_known_model_tag()` for unknown-tag warnings. `SAFETY_DENY_PATTERNS`: shared list of `(regex, description)` tuples used by both the Claude Code hook (hard block) and prompt injection (soft block for Gemini/Codex). `SAFETY_RULES`: compact 4-line hardcoded string (replaces per-pattern generated list ŌĆö reduces prompt overhead ~150 ŌåÆ ~40 tokens). `PROMPT_DAILY_LOG_TOKENS=500` (was 1500; daily log entries now 80-char first-line summaries instead of 300-char). `MIN_CAPACITY_PERCENT` env-configurable (`MIN_CAPACITY_PERCENT=10` default; override via `.env`). Per-window thresholds: `CLAUDE_FIVE_HOUR_MIN_CAPACITY_PCT`, `CLAUDE_SEVEN_DAY_MIN_CAPACITY_PCT`, `CODEX_PRIMARY_MIN_CAPACITY_PCT`, `CODEX_SECONDARY_MIN_CAPACITY_PCT`. `CAPACITY_LOG_FILE = Path(__file__).parent / "logs" / "capacity-log.md"` (moved from vault to repo `logs/`). Dev-loop timeouts: Research 3600s, Exec 7200s, Quality Review 3600s, Resolution Review 1800s. Critical-Review 3-pass: `TOOL_CR_PASS1_TIMEOUT_SEC=2400`, `TOOL_CR_PASS2_TIMEOUT_SEC=1800`, `TOOL_CR_PASS3_TIMEOUT_SEC=1800`, `TOOL_CR_PASS1_MAX_INJECT_CHARS=30000`, `TOOL_CR_MAX_PLAN_CHARS=50000`. Deep-Security-Audit: `TOOL_DSA_AGENT_TIMEOUT_SEC=1800`, `TOOL_DSA_SYNTHESIS_TIMEOUT_SEC=2400`, `TOOL_DSA_FIX_TIMEOUT_SEC=3600`, `TOOL_DSA_MAX_AGENT_OUTPUT_CHARS=15000`, `TOOL_DSA_MAX_TOTAL_INJECT_CHARS=80000`. Cleanup constants: `MEMORY_MAX_AGE_DAYS=30`, `MEMORY_ARCHIVE_DELETE_DAYS=90`, `MEMORY_DAILY_LOG_RETENTION_DAYS=30`, `MEMORY_LESSONS_RETENTION_DAYS=180`, `QUEUE_EVENTS_LOG_FILE`, `QUEUE_EVENTS_LOG_RETENTION_DAYS=30`, `QUEUE_DONE_MOVE_HOURS=48`, `QUEUE_DONE_DELETE_DAYS=7`
- **`limits.py`**: `cclimits` wrapper for provider capacity checks, OAuth refresh handling. Direct `cclimits` invocation (no npx); disk-cache via `--cache-ttl 600` limits API calls to ~6/h. **Polling-Intervalle**: `_BG_POLL_IDLE_SEC=3600` (1h, queue empty) ŌĆö `set_queue_idle(False)` wakes the bg thread on task arrival so the cache is fresh before the next task starts; the hourly idle poll only keeps the dashboard timeline warm. `_BG_POLL_AVAILABLE_SEC=300` (5 min, queue active) ŌĆö covers within-batch drift between consecutive tasks in run_once; rate_limit errors trigger `get_limits(force_refresh=True)` synchronously, so cache staleness can never delay error recovery. `set_paused(True/False)`: while paused the bg refresh thread skips all cclimits calls and only wakes on resume ŌĆö `run_watch()` calls this around its `/pause` wait so the orchestrator stops burning the monitoring quota during user-pauses. HTTP 429 resilience: (tier 0) local JSONL fallback via `claude-monitor` (no HTTP, `CLAUDE_PLAN` env var), (tier 1) snapshot cache with estimated usage tracking, Telegram notifications on 429 start/clear.
- **`logging_setup.py`**: Rotating file logger (5MB, 3 backups) + console output
- **`doctor.py`**: 16+ setup validation checks, `--fix`/`--yes` auto-repair mode. `check_model_aliases()` runs the same `_probe_model` from `heartbeat.py` against every entry in CLAUDE/GEMINI/CODEX_MODEL_ALIASES ŌĆö concurrent (ThreadPoolExecutor, `_forced_model` is `threading.local()` on the singleton providers, so per-thread model selection is safe). FAIL ŌåÆ at least one ID rejected as unknown/deprecated; WARN ŌåÆ only transient errors (rate_limit/timeout) so verification was inconclusive; PASS ŌåÆ all aliases respond live. Doctor intentionally skips Phase B (LLM-Heuristik) ŌĆö that lives only in the heartbeat to keep `--doctor` fast.
- **`shutdown.py`**: Shutdown state machine with countdown, cancellation via Telegram or new queue tasks
- **`notifier.py`**: Telegram notifications. `_truncate()` default raised to 3500 chars (was 300); task text in start/done notifications up to 300 chars (was 100)
- **`telegram_listener.py`**: Telegram bot listener. AI chat mode (`/chat`) builds context-aware prompt: SOUL.md system prompt + `memory.get_daily_context()` + user question. `_handle_message()` parses the command BEFORE the shutdown-pending check so a plain-text message that cancels a pending shutdown does NOT also get forwarded to AI chat (regression guard: `parsed_command` was previously referenced before assignment, the resulting `NameError` was swallowed by the surrounding try/except, and the message was sent to AI chat anyway).
- **`run_orchestrator.ps1`**: Crash-resistant watchdog wrapper. Compatible with both Windows PowerShell 5.1 and PowerShell 7+ ŌĆö uses `Where-Object` for crash-window filtering instead of the `[System.Predicate[datetime]]` ScriptBlock-cast (which only works in PS 7+). Restarts `python orchestrator.py --watch` on non-zero exit, exponential backoff (10s ŌåÆ 5min cap), crash-loop guard (Ōēź5 crashes / 10 min ŌåÆ 30 min cooldown + Telegram alert). Reads `TELEGRAM_BOT_TOKEN`/`TELEGRAM_CHAT_ID` from `.env` for restart notifications via `Invoke-RestMethod` (no Python dependency for alerting). `Get-EnvValue` matches Python's `_normalize_dotenv_value` rules: strips UTF-8 BOM on first line, requires whitespace before `#` for unquoted-value comments (so URLs/hashes containing `#` survive), supports quoted values with optional trailing comments. Telegram-Token is URL-encoded via `[uri]::EscapeDataString` and stripped of control chars defensively. Logs to `logs/watchdog.log` with rotation at 10 MB ŌåÆ `watchdog.log.1`. Exit 0 (Ctrl+C, /shutdown) ends the loop cleanly.
- **`tools/base_tool.py`**: `BaseTool` ABC + `_build_system_prompt(provider_name, memory_context, tool_name, cwd)` assembles system prompt with 4 memory layers ordered by stability for max prompt-cache reuse: (1) provider system prompt (SOUL+SAFETY, static), (2) curated MEMORY.md, (3) lessons (cwd-filtered, stable per tool+cwd), (4) daily log (grows during the day), (5) task-specific TF-IDF memory_context. **Lessons before Daily** is intentional ŌĆö daily grows over the day and would otherwise break the prefix cache for tool+cwd reruns. `ToolResult` carries the same four token fields as `RunResult`. `TokenCounter` helper (`.add(result)` / `.as_kwargs()`) aggregates tokens across phases ŌĆö used by review-loop and dev-loop, other tools may adopt incrementally. **`SessionContext`** helper (`SessionContext.create(provider, tool_name, cwd, cap=5)` + `.first_call_kwargs()` / `.resume_kwargs()` / `.bump()` / `.needs_rollover()` / `.rollover()`) manages CLI session lifecycle for tools that share conversation history across phases. Activates only when `provider.supports_sessions and config.CLAUDE_SESSION_ENABLED`; otherwise transparently returns empty kwargs (callers fall back to today's stateless pattern). Auto-registers UUIDs in the sidecar registry. All tool subclasses pass `tool_name=self.name, cwd=cwd` for CWD-filtered lesson injection. **`ToolTracer`** helper (`ToolTracer.create(tool_name, cwd)` + `.emit(action, **details)`) writes one JSONL event per phase/iteration/subprocess to `{cwd}/.{tool_name}/traces/{uuid}.jsonl`. Silent no-op when `cwd is None` or directory creation fails ŌĆö never breaks the tool. Suggested action vocabulary: `run_start`/`run_end`, `phase_start`/`phase_end`, `iteration_start`/`iteration_end`, `subprocess_call`/`subprocess_result`, `session_rollover`, `capacity_exhausted`. Open-ended `details` dict for tool-specific fields. Consumed by `analytics._parse_tool_traces()` for dashboard aggregation
- **`session_registry.py`**: Append-only JSONL whitelist of orchestrator-created Claude session UUIDs at `logs/orchestrator-sessions.jsonl`. `register_session(uuid, tool, cwd)` appends an entry; `is_orchestrator_session(uuid)` checks membership; `prune_old(retention_days)` splits into kept/expired and rewrites the registry. Used by the heartbeat session-cleanup handler to distinguish orchestrator sessions from interactive Claude Code sessions (which we MUST NOT delete) ŌĆö both live under `~/.claude/projects/<encoded-cwd>/<uuid>.jsonl`
- **`tools/research_qa.py`**: `ResearchQATool` ŌĆö 3-phase read-only pre-implementation workflow (Discovery ŌåÆ Analysis ŌåÆ Questions). Output to `{cwd}/.research-qa/`. Registered as `#tool:research-qa`
- **`tools/review_loop.py`**: `ReviewLoopTool` ŌĆö iterative code review fixing ALL P1/P2/P3 findings (max 20 iterations). Infinite-loop detection via finding signature dedup. Fix prompt is split into stable prefix (`_FIX_PROMPT_STABLE`, cached across iterations) + volatile suffix (`_FIX_PROMPT_VOLATILE` with iteration-specific findings + lessons hint at the end) ŌĆö Anthropic prompt cache matches the longest identical prefix, so volatile content lives at the end. **Phase B sessions**: `SessionContext` shares conversation history across reviewŌåÆfixŌåÆreview iterations with `cap=5` rollover; review prompt always reads `git diff` fresh, so a fresh-session continuation is seamless. Fallback on `session_missing` resets to a fresh UUID and retries once. **Second-Opinion** (opt-in via `#second_opinion:<alias>` tag, e.g. `#second_opinion:or_glm`): in iteration 1 only, after the primary review, a non-agentic LLM (typically OpenRouter via `or_*` alias) receives the pre-loaded git diff + the primary's findings and lists *additional* P1/P2/P3 issues. Resolved via `_resolve_second_opinion(alias)` which maps model aliases to their owning provider (or_* ŌåÆ openrouter, claude_* ŌåÆ claude, etc.) and uses `dispatcher.get_provider_by_name`. Extra findings are union-merged into the fix prompt (exact-string dedup). Non-fatal skips: SO provider in cooldown, diff fetch fails, diff > `TOOL_RL_SECOND_OPINION_MAX_DIFF_CHARS` (30k), or `_resolve_second_opinion` returns None ŌĆö primary findings continue alone. Fix phase stays on Claude (or whatever the primary provider is) ŌĆö second-opinion is read-only. Timeout: `TOOL_RL_SECOND_OPINION_TIMEOUT_SEC` (10 min).
- **`tools/dev_loop.py`**: `DevLoopTool` ŌĆö Research+Plan ŌåÆ Execute ŌåÆ Quality-Review + Resolution-Review (`RESOLVED`/`PARTIAL`/`UNRESOLVED`) loop until both reviews pass (max 20 iterations). **Phase 1 (Research) and Phase 1.5 (Plan) are merged into a single subprocess call** (`_RESEARCH_AND_PLAN_PROMPT` produces both sections; saves ~42 k cache_creation tokens vs. the prior two-call pattern). Skip-mode (`policy.tool_phases.dev-loop.plan_approval=skip`) falls back to research-only via `_RESEARCH_ONLY_PROMPT`. State persisted in `{cwd}/.dev-loop/state.json` under phase `research_and_plan_done` (or `research_done` for skip-mode), enabling capacity-exhausted retry without re-running the merged phase. **Phase B sessions**: `SessionContext` shares conversation history across all phases when enabled, with `cap=5` rollover (fresh UUID after every 5 iterations to bound conversation length ŌĆö explicit findings re-injection in `review_context` keeps the model in sync across the boundary). Fallback on `error="session_missing"` (e.g. cleanup deleted JSONL between calls): rollover + retry once with the same prompt. Registered as `#tool:dev-loop`
- **`tools/critical_review.py`**: `CriticalReviewTool` ŌĆö 3-pass adversarial review. Pass 1: radical-honesty analysis. Pass 2: adversarial challenge by different persona. Pass 3 (Synthesis): produces improved plan as `{name}-v2.md` (only when plan file referenced in task). Plan file resolution: file paths in CWD ŌåÆ wikilinks in vault. ## Phase B Feature Flag ŌĆö `CLAUDE_SESSION_ENABLED`

CLI-level conversation sessions on the Claude provider are gated by `config.CLAUDE_SESSION_ENABLED` (default **OFF**, opt-in via `.env`). When enabled:
- Tools that allocate a `SessionContext` get a UUID and pass `--session-id`/`--resume` on subsequent `claude --print` calls.
- Anthropic prompt-cache hits go from "static system-prompt only" to "full conversation history" ŌĆö projected ~30-50 % token savings on multi-phase tools (dev-loop, review-loop).
- All UUIDs are registered in `logs/orchestrator-sessions.jsonl` (sidecar) so the heartbeat session-cleanup never deletes interactive Claude Code sessions.
- Fresh sessions every `cap=5` iterations cap conversation length (Sonnet 200 k context limit, Opus 1 M).
- On `error="session_missing"` (e.g. cleanup race) tools rollover and retry once.

**Rollback**: set `CLAUDE_SESSION_ENABLED=false` in `.env` and restart the orchestrator. ClaudeProvider then ignores `session_id`/`resume` parameters and falls back to today's stateless `claude --print` invocation. No code revert needed.

`config.ORCH_SESSION_RETENTION_DAYS=14` controls how long expired registry entries live before the cleanup deletes them.

Codex and Gemini providers also have CLI-level resume capabilities, but `supports_sessions=False` keeps them on stateless behaviour until empirical evidence justifies the implementation cost.

## Key Patterns

- **Singletons with threading**: `PolicyEngine`, `UsageSuggester`, providers ŌĆö each has own `_lock`, `threading.Event` for blocking operations, no global mutex
- **Provider-bound model tags**: `#claude_opus`, `#gemini_flash`, `#codex_mini`, `#or_minimax_free`, `#or_glm` etc. are resolved via `config.model_id_for_provider(tag, provider_name)`, which returns `None` if the tag doesn't belong to the target provider. This prevents a Claude model ID from leaking to Gemini during provider fallback. The `_forced_model` attribute lives on `BaseProvider` (via `threading.local()`), applied to any provider. Orchestrator and `parallel_runner` set it per-task without a claude-only gate. Cross-provider tools (e.g. `critical_review.py` pass2) re-apply the tag against the pass2 provider separately ŌĆö see `CriticalReviewTool.run()` around `pass2_provider.run(...)`.
- **OpenRouter never in fallback chain**: `dispatcher._PRIORITY` deliberately omits `openrouter` so untagged tasks never accidentally route to a pay-per-token provider. Tag resolution uses `_providers.get(name)` (not `[name]`) so an `#or_*` tag without `OPENROUTER_API_KEY` configured silently falls back to claude/gemini/codex instead of raising KeyError. Same pattern would extend cleanly to future opt-in providers.
- **Mtime-cached config**: Policy, profiles, SOUL.md, heartbeat all use `(mtime, content)` tuple caching for hot-reload in `--watch` mode
- **Token-budget injection**: `_build_prompt()` truncates skill/memory/wikilink context to `PROMPT_*_TOKENS` constants before assembly
- **Sidecar file locking**: `queue_manager.py` uses `.lock` file with platform-specific locking for multi-process safety
- **Subtask-aware queue mutations**: `mark_done/mark_retry/finalize_task_with_result` accept `subtasks` kwarg; `_replace_open_task_line` uses it to disambiguate duplicate task texts in parallel queues. Fallback re-scan is O(N) ŌĆö skips subtask block scan for non-matching task lines.
- **`task_subtasks` in run_once**: Extracted via `getattr(queue_task, "subtasks", None)` at loop start for test-mock compatibility (some tests use bare `SimpleNamespace`).
- **Task dependencies (`#id:`/`#needs:`)**: `#id:name` tags a task with a unique ID. `#needs:name1,name2` blocks a task until all named deps appear as `[x]` or `[-]` in the file. `_collect_completed_ids()` scans the full file for done/failed tasks. Two-pass in `read_queue_items()` ŌĆö short-circuit if no `#needs:` present. Blocked tasks keep `QueueTask.blocked_reason != ""` and are skipped by `run_once()` (no `mark_done` ŌåÆ stays in queue for next cycle). Queue header shows `(N ausf├╝hrbar, M blockiert)` when any tasks are blocked.
- **`.env` comment stripping**: `_normalize_dotenv_value()` requires whitespace before `#` for unquoted values (protects URLs/paths containing `#`); quoted values allow `#` anywhere after the closing quote.
- **HTTP 429 resilience**: When `cclimits` monitoring API returns 429, `limits.py` retries with backoff (5s/10s), then applies 3-tier fallback: (0) local JSONL via `claude-monitor` (`_get_claude_limits_from_local`, `CLAUDE_PLAN` env var, uses `token_counts.total_tokens`), (1) snapshot cache with estimated usage tracking (`_429_base_snapshot`, `_429_estimated_usage`), (2) optimistic cold-start. Polls back off to 5 minutes. `report_estimated_usage()` called after each task. State resets when 429 clears. Disk-cache (`--cache-ttl 600`) reduces normal API calls.
- **3-tier token estimation**: `estimate_task_usage_pct()` in `limits.py` ŌĆö (1) actual token counts from Claude JSON output, (2) text-based estimate from prompt/output char lengths, (3) duration heuristic fallback. Configured per provider via `ESTIMATE_TOKENS_PER_PCT`.

## Safety Rules (enforced in code)

- **Hard deny (Claude Code hook)**: `scripts/safety_hook.py` as `PreToolUse` hook blocks dangerous Bash commands before execution ŌĆö works even in `--dangerously-skip-permissions` mode. Patterns defined in `config.SAFETY_DENY_PATTERNS`
- **Soft deny (prompt injection)**: `SAFETY_RULES` text (auto-generated from same patterns) injected into all provider prompts via `SYSTEM_PROMPTS` ŌĆö covers Gemini/Codex which have no hook system
- **Blocked categories**: `rm -rf`, `git push --force/-f`, `git reset --hard`, `git clean -f`, `git checkout -- .`, `DROP/TRUNCATE TABLE`, `DELETE FROM` without WHERE, `format`/`mkfs`/`diskpart`, fork bombs, raw disk writes, credential exfiltration via curl/wget, Windows `del /s`, `rd /s /q`, `Remove-Item -Recurse -Force`
- CWD validation against `ALLOWED_CWD_ROOTS` ŌĆö rejects relative paths and parent escapes
- Skill gating checks requirements (bins, env vars, OS, provider) before execution
- Policy layer can block tasks pending Telegram approval
